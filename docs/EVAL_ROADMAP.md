# 评测层路线图：有潜力但尚未落地

记录来自 CoRE / VSRM / LLM-as-a-Judge 文献中"值得做但当前未做"的项，及其**阻塞前提**与
**建议接入点**。已落地的部分见 `docs/EVAL.md`。

优先级图例：⭐⭐⭐ 拿到前提就该上 · ⭐⭐ 有价值可排期 · ⭐ 视情况。

---

## 1. 真人金标准校准（Calibration）　⭐⭐⭐
**做什么**：离线集混入公司真实"金牌人工客服"对话，让裁判**盲测**给所有对话（AI + 真人）打分；
若 AI 自然度被判高于真人，说明裁判"偏好机器味"，据此回调裁判 prompt。
**为什么最重要**：这是把 LLM-as-judge 从"自说自话"变可信的唯一硬校准，直接对冲下面第 2 条的
自我偏好陷阱。
**阻塞前提**：需要一批真实历史人工售前对话（邮件 / WhatsApp / 展会跟单）。**目前缺这批数据。**
**落地建议**：
- 用**成对(pairwise)**而非绝对打分做校准最稳；判据用**秩相关(Kendall τ / Spearman)**，
  不要用"AI 不得超真人"的单点阈值。
- 金标集切 **calibration / holdout** 两份，防止把裁判过拟合到那几条真人对话。
- 复用 `judges.judge_pairwise()` + `validate_judge.run_position_swap()`（已支持 AB 交换）。

## 2. 异构裁判 / 多模型陪审团（Jury, Majority Vote）　⭐⭐⭐
**做什么**：用 ≥2 个不同架构的模型（如 GPT-4o / Claude / Qwen）各判一次，取多数表决或均值。
**为什么**：不同模型偏见不同（GPT 易接受推测、Mistral/Llama 重客观证据），投票中和单点偏见；
更关键——**我们大脑与默认裁判都是 MiniMax**，等于"自己判自己"，文献证明裁判会偏袒**低困惑度**
文本（即平庸八股），这正是自然度评测的头号敌人。
**阻塞前提**：需要第二个异构模型的 API key（当前只有 MiniMax）。
**落地建议**：`config.EVAL_JUDGE_MODEL` 钩子已就位——填异构模型即可让裁判≠生成模型；
陪审团则在 `score_llm` 外套一层多模型循环 + 投票聚合。

## 3. G-EVAL：Token 概率加权连续打分　⭐（当前**被阻塞**）
**做什么**：不取离散整数分，而是提取输出分数 token 的概率做加权求和，得到连续、低平局的分数。
**阻塞前提**：需要 API 返回 logprobs。**已实测：MiniMax OpenAI 兼容接口忽略 `logprobs` 参数（返回 None），
此路不通。** 换到支持 logprobs 的裁判模型（多数 OpenAI 系支持）后再启用。
**替代**：当前用 1-3 低基数 + 四镜头多轴，已在很大程度上缓解"平局/低方差"。

## 4. 多轮 CoRE 辩论委员会　⭐⭐
**做什么**：四角色（语言学家 / 普通用户 / HCI / 领域专家）多轮相互辩论后给分，而非一次打分。
**为什么暂缓**：成本随 N×角色×轮次爆炸；且"辩论优于单个好 rubric"的证据是混合的。**当前已用
"单次多视角 rubric"（Combined Budget：CoT+rubric 一次调用）拿到大部分价值。**
**触发条件**：仅当第 1 条校准发现**单裁判压不住机器偏好**、单次 rubric 无法对齐真人排序时，
再升级为真·多轮辩论。避免为可能不需要的复杂度预付。

## 5. 动态对抗 / 众包盲测（Chatbot-Arena 式）　⭐（离线场景不适用）
**做什么**：引入真实用户提最新问题、匿名配对盲测，抗数据污染与过拟合。
**为什么暂不做**：无线上流量、无众包渠道——这正是纯离线场景做不了的。
**我们的替代**：`sim_customer`（多样画像动态生成）+ 真人金标准校准，作为离线保真替身。
未来有线上流量后可引入配对盲测埋点。

## 6. 数据污染 / 静态集过拟合防护　⭐⭐
**做什么**：seed 集会随迭代被"背题"。定期刷新 seed 词表、扩充画像维度、保留 holdout 私有集。
**落地建议**：`personas.py` 词表与维度可扩展；建立"公开回归集 + 私有 holdout 集"两层。

## 7. 自然度轴的确定性扩展　⭐⭐
**做什么**：把更多"AI 味"信号从 LLM 下沉到规则（更快更稳）。候选：句式重复率 / n-gram 自相似、
客套语密度、每轮信息密度（新信息 token / 总 token）、表情堆砌。
**落地建议**：并入 `naturalness.py`，与现有 Markdown/机械词/思维泄漏/冗长同级。

## 8. 复合记分卡与红线门禁　⭐⭐
**做什么**：把 task-success（参数正确率、风险路由）与 naturalness 合成一张排名卡，但
**任务正确性/风险路由设为红线**——自然度再高，红线不达标直接判负，防止选出"很会聊天但问错参数、
放过制裁国"的模型。
**落地建议**：在 `run_eval.aggregate` 之上加一层加权 + 红线过滤的 `rank()`。
